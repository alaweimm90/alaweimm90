## Version Format

We use [Semantic Versioning](https://semver.org/):

- **MAJOR version** when making incompatible changes to documentation structure
- **MINOR version** when adding new features (new docs, new scripts, new templates)
- **PATCH version** when making bug fixes or improvements

## Categories

- **Added** for new features, documentation, templates, or scripts
- **Changed** for changes in existing documentation or scripts
- **Deprecated** for soon-to-be removed features or documentation
- **Removed** for now removed features
- **Fixed** for any bug fixes
- **Security** for security-related updates
