## Common Warnings

### Workflow Security Hardening

Many workflows are missing the `permissions` block. This is a security best practice to limit token scope.

**Affected workflows include:**
- `deploy.yml` (multiple repos)
- `policy.yml` (multiple repos)
- `reusable-policy.yml` (multiple repos)
- `mezan-ci.yml`
- `talair-ci.yml`

**Fix**: Add explicit permissions block to each workflow:
```yaml
permissions:
  contents: read
```

### Docker Best Practices

Several Dockerfiles are missing `HEALTHCHECK` directives:
- `AlaweinOS/fitness-app/Dockerfile`
- `AlaweinOS/QAPlibria-new/Dockerfile`
- `AlaweinOS/MEZAN/Libria/libria-meta/Dockerfile`

**Fix**: Add HEALTHCHECK to Dockerfiles:
```dockerfile
HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost:8080/health || exit 1
```

### Continue-on-Error Anti-Pattern

Multiple workflows use `continue-on-error: true` which may hide failures:
- `mezan-ci.yml` (20+ instances)
- `talair-ci.yml`
- Various deploy workflows

**Recommendation**: Remove `continue-on-error: true` or use it only for non-critical optional steps.
